WSO2 APIM Publisher — Federated SSO Login Flow

Source: mpa.har · APIM :9443 (Embedded IS) federated via OIDC to External IS :9444 · click any step to expand

🔗 Federated Architecture:  APIM Embedded IS :9443 acts as a Service Provider (SP) — it delegates authentication to an external Identity Provider. The External WSO2 IS :9444 acts as the Identity Provider (IdP) — it authenticates the user and issues an OIDC code back to APIM. APIM's embedded IS exchanges that code for user identity, then issues its own authorization code to the Publisher SPA.
Key difference vs non-federated:  Step 4 triggers a second OAuth2 Authorization Code flow nested inside the first — browser traverses two /oauth2/authorize endpoints and two consent pages before tokens are issued. The External IS also shows a consent screen (scope: openid) for the wso2am service provider.

Federated vs Non-Federated — Key Differences
NON-FEDERATED (logintrace.har)
Actors: Browser + APIM(:9443 only)
Steps 4–7: Embedded IS handles auth (BasicAuthenticator)
Consent: Skipped (saasApp=true)
Redirects: 4 (login→IS→commonauth→IS→callback)
Cookies during auth: commonAuthId set on :9443 only
/oauth2/authorize calls: 2 (Hit 1 + Hit 2)
FEDERATED (mpa.har) — this diagram
Actors: Browser + APIM(:9443) + External IS(:9444)
Steps 5–10: External IS authenticates (OIDCAuthenticator delegates to :9444)
Consent: Shown on :9444 for scope=openid (wso2am SP)
Redirects: 8+ (nested OAuth2 Code flow)
Cookies during auth: commonAuthId on :9444 AND :9443 (two sessions)
/oauth2/authorize calls: 4 (2 on :9443 + 2 on :9444)
Back-channel token exchange: :9443 → :9444 /oauth2/token (hidden from browser)
Token Split Strategy — XSS Mitigation (same as non-federated)
access_token = WSO2_AM_TOKEN_1_Default (P1) + AM_ACC_TOKEN_DEFAULT_P2 (P2, HttpOnly)
refresh_token = WSO2_AM_REFRESH_TOKEN_1_Default (P1) + AM_REF_TOKEN_DEFAULT_P2 (P2, HttpOnly)
id_token = AM_ID_TOKEN_DEFAULT_P1 + AM_ID_TOKEN_DEFAULT_P2 (both path: /publisher/services/logout)

Neither half alone is a valid token. XSS can read P1 but cannot use it without P2 (HttpOnly). login_callback.jsp, introspect.jsp, refresh.jsp always reassemble P1+P2 server-side.
