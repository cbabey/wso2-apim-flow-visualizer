ACTUAL SOURCE — EXTRACTED FROM index.bundle.js
// ProtectedApp.componentDidMount()
const user = AuthManager.getUser();          // ← check in-memory store first
if (user) {
  // Same-tab refresh: user already in memory
  user.scopes.includes('apim:api_view') && user.scopes.includes('apim:publisher_settings')
    ? setState({ user, userResolved: true })     // ✓ render dashboard
    : setState({ notEnoughPermission: true });    // ✗ 403
} else {
  // New tab / hard refresh: try to restore from cookies
  AuthManager.getUserFromToken()
    .then(user => setState({ user, userResolved: true }))  // ✓ dashboard
    .catch(err => setState({ userResolved: true }));       // ✗ → login redirect
}

// getUserFromToken() — the actual cookie check:
const tokenP1 = Utils.getCookie("WSO2_AM_TOKEN_1_Default");
if (!tokenP1) return reject(new Error("NO_TOKEN_FOUND"));  // no network call
return fetch("/publisher/services/auth/introspect", { credentials: "same-origin" })
  .then(r => r.json())
  .then(result => {
    if (!result.active) throw new Error("INVALID_TOKEN");
    const user = new User(env.label, result.username);
    user.scopes = result.scope.split(' ');
    AuthManager.setUser(user);   // store in memory for next getUser() call
    return user;
  });

Token Split Strategy — XSS Mitigation
access_token = WSO2_AM_TOKEN_1_Default (P1) + AM_ACC_TOKEN_DEFAULT_P2 (P2, HttpOnly)
refresh_token = WSO2_AM_REFRESH_TOKEN_1_Default (P1) + AM_REF_TOKEN_DEFAULT_P2 (P2, HttpOnly)
id_token = AM_ID_TOKEN_DEFAULT_P1 + AM_ID_TOKEN_DEFAULT_P2 (both path: /publisher/services/logout)

Neither half alone is a valid token → XSS in the SPA can read P1 but cannot use it without P2 (HttpOnly). Server-side JSPs (login_callback.jsp, introspect.jsp, refresh.jsp) always reassemble P1+P2 server-side.
Embedded IS — Key DB Tables & Caches
DATABASE TABLES
IDN_OAUTH_CONSUMER_APPS — OAuth2 app registration (client_id, secret, callback)
SP_APP + SP_INBOUND_AUTH — Service Provider config (saasApp, skipConsent flags)
IDN_OAUTH2_SCOPE — scope definitions
IDN_OAUTH2_SCOPE_BINDING — role → scope permission map
IDN_OAUTH2_AUTHORIZATION_CODE — issued codes (hashed, PKCE fields, 300s TTL)
IDN_OIDC_SESSION_STATE — OIDC session participants per browser (opBrowserState)
IDN_OPENID_USER_RPS — user ↔ app approval records (consent skip check)
HAZELCAST CACHES
SessionDataCache → keyed by sessionDataKey — parks OAuth2Parameters during auth redirect
AuthnResultCache → keyed by sessionDataKey — holds AuthenticationResult after /commonauth
AuthorizationGrantCache → keyed by authCode — carries user attributes + amr + pkce to /oauth2/token
AppInfoCache → keyed by clientId — caches OAuthAppDO to avoid repeat DB lookups
OAuthCache → keyed by clientId:code — fast code lookup at token exchange
sessionDataKey lifecycle: Generated in Hit 1 → passed through browser as correlation ID → used to restore state in Hit 2 → immediately invalidated (single-use). Replaying a consumed key fails with "Context does not exist / invalidated cache".
